# BG-05: Утренний дайджест

Тег: BG-05

Эпик: EPIC-05. Celery, Redis и фоновые рассылки

Зависимости: CORE-03, TG-01, BG-01

## Зачем этот тикет

Каждое утро в **09:00 MSK** пользователь получает список активных задач на сегодня. Пустой список — **не** слать сообщение.

## Технологии и контекст

- Инфраструктура Celery/Beat уже в **BG-01** — здесь добавляем **новый job** и регистрируем его в том же Beat schedule.
- Бизнес-логика выбора задач — через `TaskService` / CORE-03, не в Celery task.
- Константа: `MORNING_DIGEST_HOUR` (default 9) в settings.
- Документация: `tz/MAILING_PIPELINES.md` § «Утренний дайджест», UC-06.

## Границы тикета

**Входит:** task утреннего дайджеста, регистрация в Beat (09:00), формат сообщения, защита от повтора за календарный день.

**Не входит:** due reminders (BG-01), вечерняя проверка (BG-06), полная идемпотентность с claim (BG-F01), retry policy (BG-F02).

## Правило: `TaskEvent` (если фиксируете отправку)

| Действие | `event_type` | `message_id` |
| --- | --- | --- |
| Отправлен утренний дайджест | `digest_sent` | опционально |

Если в MVP достаточно логов без `TaskEvent` — согласуй с тимлидом; в `decomp.md` / DB-04 тип `digest_sent` уже зарезервирован.

## Что реализуется

1. **Task `send_morning_digest`** (имя на усмотрение):
   - для каждого user с активными задачами **на сегодня** (в `DEFAULT_TIMEZONE`) — собрать список;
   - если список пуст — **skip**, не слать «пустое» сообщение;
   - `TelegramSender.send_*` — текст списка (формат на усмотрение, читаемый в Telegram);
   - **не дублировать** дайджест тому же user за тот же календарный день (простая проверка: `TaskEvent(digest_sent)` за сегодня или отдельное поле — на выбор, без industrial lock).
2. **Beat schedule** — cron/every day at `MORNING_DIGEST_HOUR` в `DEFAULT_TIMEZONE`.
3. Подключить job в существующий `config/celery.py` из BG-01.

## Примеры для ручной проверки

| # | Подготовка | Действие | Ожидание |
| --- | --- | --- | --- |
| 1 | 2–3 active задачи на сегодня + 1 без даты + 1 на завтра | Trigger job (beat или manual call) | Одно сообщение: только задачи **на сегодня** |
| 2 | Нет задач на сегодня | Trigger job | Сообщение **не** отправляется |
| 3 | Дайджест уже отправлен сегодня | Повторный trigger | Второе сообщение **не** уходит |
| 4 | Задача `done`/`cancelled` | Trigger job | Не попадает в список |

## Кривые кейсы — учесть

| Ситуация | Ожидание MVP |
| --- | --- |
| Задача создана в 08:59 на сегодня | Попадает в дайджест 09:00 |
| Задача только с датой без времени | Правило «на сегодня» — согласовать с CORE-03 / tables.md |
| User без задач на сегодня, но с undated | Дайджест не шлём (UC-06 — список на сегодня) |

## Acceptance criteria

- Job зарегистрирован в Beat на 09:00 MSK (или `MORNING_DIGEST_HOUR`)
- Отправляется список active задач user'а на сегодня
- Пустой список → нет сообщения
- Повтор за тот же день → нет дубля
- Бизнес-логика в сервисе, Celery task — тонкая обёртка

## Ручная проверка разработчиком

- [ ] Подготовить задачи на сегодня / не на сегодня (строка 1 таблицы)
- [ ] Trigger job → сообщение с корректным списком
- [ ] Без задач на сегодня → тишина (строка 2)
- [ ] Повторный trigger в тот же день → без дубля (строка 3)
- [ ] Зафиксировать в комментарии к тикету: время trigger, скрин/текст сообщения, сверка с БД
