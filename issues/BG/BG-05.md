# BG-05: Утренний дайджест

Тег: BG-05

Эпик: EPIC-05. Celery, Redis и фоновые рассылки

Зависимости: CORE-03, TG-11, BG-01, TG-05

## Зачем

Каждое утро в **09:00 MSK** пользователь получает **карточки** активных задач на сегодня. Пустой список — не слать. Одна карточка на задачу (TG-11, variant `digest`).

## Технологии

- Celery/Beat из BG-01
- `TaskService.list_for_day(user, today)` — CORE-03
- `TelegramSender.send_task_card(chat_id, task, variant="digest")` — TG-11
- `MORNING_DIGEST_HOUR` (default 9), `DEFAULT_TIMEZONE`

## Правило: `TaskEvent`

| Действие | `event_type` | `message_id` |
| --- | --- | --- |
| Отправлена карточка дайджеста | `digest_card_sent` | да |

## Что реализуется

1. **Task `send_morning_digest`**:
   - для каждого user — задачи на сегодня (`active`);
   - пустой список → skip;
   - на каждую задачу — `send_task_card(variant="digest")`;
   - идемпотентность: если у user уже есть `digest_card_sent` за сегодня → skip всего дайджеста.
2. **Beat** — ежедневно `MORNING_DIGEST_HOUR` в `DEFAULT_TIMEZONE`.
3. Регистрация в `config/celery.py`.

## Границы

**Входит:** job, Beat, карточки, идемпотентность за день.

**Не входит:** due reminders (BG-01), вечер (BG-06), callbacks (TG-05), reply (TG-07), claim (BG-F01).

## Acceptance criteria

- 09:00 MSK — карточки только задач на сегодня
- N задач → N карточек с «Сделано» / «Удалить» + footer Reply
- Пустой день → тишина
- Повтор за тот же день → без дубля
- Бизнес-логика в сервисе, Celery task — обёртка

## Ручная проверка

- [ ] 2–3 задачи на сегодня → 2–3 карточки
- [ ] Задача на завтра / undated — не в дайджесте
- [ ] Повторный trigger — без дубля
- [ ] Reply на digest-карточку → перенос (TG-07)
- [ ] «Удалить» на карточке → `deleted` (TG-05)
