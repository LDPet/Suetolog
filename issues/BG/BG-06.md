# BG-06: Вечерняя проверка и вопрос о переносе

Тег: BG-06

Эпик: EPIC-05. Celery, Redis и фоновые рассылки

Зависимости: CORE-03, CORE-05, TG-07, BG-01

## Зачем этот тикет

В **20:00 MSK** бот спрашивает пользователя о переносе задач, по которым reminder был отправлен, но **не было реакции** ✅/❌. Ответ — **Reply** с новой датой (обрабатывает TG-07).

## Технологии и контекст

```
20:00 Beat → evening job → найти reminders без реакции → Telegram вопрос → TaskEvent(evening_question_sent)
Пользователь Reply → TG-07 → find_by_message_id → reschedule + TaskEvent(rescheduled)
```

- Зависит от **BG-01** (reminder с `message_id`) и **TG-07** (lookup по `message_id`).
- Константа: `EVENING_MISSED_CHECK_HOUR` (default 20).
- Документация: `tz/MAILING_PIPELINES.md`, UC-10.

## Границы тикета

**Входит:** evening job, выбор кандидатов, отправка вопроса, `TaskEvent(evening_question_sent, message_id)`, защита от повтора вопроса по тому же reminder/дню.

**Не входит:** парсинг даты из Reply (TG-07 + AI-04), claim/lock (BG-F01), retry policy (BG-F02).

## Правило: `TaskEvent`

| Действие | `event_type` | `message_id` |
| --- | --- | --- |
| Отправлен вечерний вопрос о переносе | `evening_question_sent` | да |
| Пользователь ответил новой датой | `rescheduled` | null |

`evening_question_sent` пишет **mailing service / Celery job**; `rescheduled` — **TaskService** в TG-07.

## Что реализуется

1. **Task `send_evening_missed_check`** (имя на усмотрение):
   - найти reminders: отправлены (`sent_time` есть), задача still `active`, **нет** реакции (статус задачи не `done`/`cancelled`);
   - для каждого кандидата — отправить вопрос «перенести?» (текст на усмотрение);
   - сохранить `TaskEvent(evening_question_sent, message_id=...)`;
   - **не задавать** тот же вопрос повторно (проверка по существующему event или флагу).
2. **Beat schedule** — каждый день в `EVENING_MISSED_CHECK_HOUR`.
3. Reply-flow **не** реализовывать здесь — только убедиться, что `message_id` совместим с `find_by_message_id` в TG-07.

## Примеры для ручной проверки

| # | Подготовка | Действие | Ожидание |
| --- | --- | --- | --- |
| 1 | Reminder отправлен (BG-01), реакции нет | Trigger evening job | Вопрос в Telegram; `TaskEvent(evening_question_sent, message_id)` |
| 2 | Пользователь поставил ✅ до 20:00 | Trigger job | Вопрос **не** отправляется |
| 3 | Вопрос уже отправлен сегодня по этому reminder | Повторный trigger | Дубль вопроса **не** уходит |

## Кривые кейсы — учесть

| Ситуация | Ожидание MVP |
| --- | --- |
| Reminder отправлен, пользователь удалил задачу | Не слать вопрос |
| Несколько reminders без реакции | Отдельный вопрос на каждый (или одним сообщением — зафиксируй в реализации) |
| Reply не на то сообщение | TG-07: подсказка, без переноса |

## Acceptance criteria

- Job в Beat на 20:00 MSK
- Вопрос только по active reminders с `sent_time` и без реакции
- `TaskEvent(evening_question_sent, message_id)` при отправке
- Повтор job не дублирует вопрос
- Reply с датой обрабатывается TG-07 (ручная проверка end-to-end)

## Ручная проверка разработчиком

- [ ] Reminder без реакции → вечерний вопрос (строка 1)
- [ ] В БД: `TaskEvent(evening_question_sent, message_id)` — тот же id, что в Telegram
- [ ] После ✅ — вопрос не приходит (строка 2)
- [ ] Повторный trigger — без дубля (строка 3)
- [ ] Reply «завтра в 10:00» → перенос задачи, `TaskEvent(rescheduled)` (строка 4, TG-07)
