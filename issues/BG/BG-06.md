# BG-06: Вечерняя проверка и вопрос о переносе

Тег: BG-06

Эпик: EPIC-05. Celery, Redis и фоновые рассылки

Зависимости: CORE-03, CORE-05, TG-07, TG-11, BG-01, TG-05

## Зачем

В **20:00 MSK** бот спрашивает о переносе **активных задач с `due_to` на сегодня**, по которым пользователь **ещё не нажал «Сделано» / «Удалить»**. Ответ — Reply с новой датой (TG-07). Карточка — TG-11, variant `evening`.

## Поток

```
20:00 → evening job → кандидаты → send_task_card(evening) → TaskEvent(evening_question_sent, message_id)
Reply → TG-07 → reschedule → TaskEvent(rescheduled)
```

## Критерий кандидата

- `Task.status = active`
- `due_to` приходится на **сегодня** (локальный день)
- Вечерний вопрос по этой задаче сегодня ещё **не** отправляли (`evening_question_sent` за день)

Факт отправки Reminder **не** проверяем — ориентир на срок задачи, не на `sent_time`.

**Не используем:** реакции ✅/❌ (TG-08 superseded).

## Правило: `TaskEvent`

| Действие | `event_type` | `message_id` |
| --- | --- | --- |
| Отправлен вечерний вопрос | `evening_question_sent` | да |
| Reply с новой датой | `rescheduled` | null (пишет TG-07 / TaskService) |

## Что реализуется

1. **Task `send_evening_missed_check`** — выбор кандидатов по `due_to` на сегодня, `send_task_card(variant="evening")`, `TaskEvent(evening_question_sent, message_id)`.
2. **Beat** — `EVENING_MISSED_CHECK_HOUR` (default 20).
3. Reply **не** здесь — только совместимость `message_id` с TG-07.

## Acceptance criteria

- Job в Beat 20:00 MSK
- Карточка с «Сделано» / «Удалить» + footer Reply
- Active задача с `due_to` на сегодня без evening_question_sent → вопрос приходит (даже без Reminder)
- Задача на другой день / без `due_to` / закрытая — вопрос не приходит
- Повтор job — без дубля вопроса
- Reply end-to-end с TG-07

## Ручная проверка

- [ ] Active задача на сегодня → вечерняя карточка, `evening_question_sent` + `message_id`
- [ ] «Сделано» до 20:00 → вопрос не приходит
- [ ] Reply «завтра в 10» → `rescheduled`
- [ ] Повторный trigger — без дубля
