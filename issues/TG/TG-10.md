# TG-10: Сквозная проверка use-cases MVP (обязательный)

Тег: TG-10

Эпик: EPIC-03. Telegram-бот и пользовательские сценарии

Зависимости: TG-03, TG-05, TG-07, TG-08, TG-09, CORE-07, BG-01, BG-06

**Волна 6** — финал, после сборки системы.

## Не follow-up

Закрывается, когда все потоки сдали свои тикеты и каждый разработчик уже прогнал самопроверку в своём TG-тикете. TG-10 — общий финальный прогон **всей** системы в Telegram.

## Что делает исполнитель

- Поднимает полный стек локально: `runbot`, worker, beat, PostgreSQL, Redis
- Прогоняет сценарии из `use_cases.md` (MVP-часть)
- Фиксирует результат в трекере: что работает, что сломано, скрин/текст сообщений
- Найденные баги — **отдельные тикеты**, TG-10 не закрывают с известными блокерами

## Чеклист use-cases

| UC | Сценарий | Проверить |
| --- | --- | --- |
| UC-01 | Первое обращение | `/start`, User в БД |
| UC-02 | Создание задачи голосом | voice с датой → Task + Reminder + подтверждение |
| UC-04 | Задача без даты | voice «без даты» → Task без Reminder |
| UC-05 | Список без даты | `/undated`, карточки, кнопки |
| UC-06/07 | Назначение даты / удаление | Reply с датой; удаление кнопкой |
| UC-09 | Due reminder | Celery → сообщение в Telegram, `message_id` в БД |
| UC-10/11 | Реакции | ✅ → `done`; ❌ → `cancelled` |
| UC-12 | Вечерний перенос | вопрос в 20:00 → Reply → `rescheduled` |

## Acceptance criteria

- Все пункты чеклиста пройдены или заведены баги с приоритетом
- Есть краткий отчёт в трекере по таблице UC выше — финальный E2E-артефакт (отдельный QA-03 не нужен)
- Критический путь demo работает: **голос → задача → reminder → реакция**
- TG-10 блокирует DOC-03 (demo video) и финальную сдачу

## Ручная проверка

Весь тикет — одна большая ручная проверка; отдельный TG-QA не нужен.
