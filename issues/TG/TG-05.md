# TG-05: `/undated` + inline-кнопки (merged TG-05/06)

Тег: TG-05

Эпик: EPIC-03. Telegram-бот и пользовательские сценарии

Зависимости: TG-01, CORE-03

## Правило: `TaskEvent` на каждое действие с задачей

Любое действие, которое **меняет задачу** или **отправляет по ней сообщение бота**, создаёт `TaskEvent`. Событие пишет **сервисный слой** (`TaskService` / CORE-03), handler только вызывает сервис.

| Действие | `event_type` | `message_id` |
| --- | --- | --- |
| Отправлена карточка `/undated` | `undated_card_sent` | да |
| Удаление через кнопку | `deleted` | null |

## Что реализуется

- `reminder/bot/handlers/undated.py`:
  - handler на `/undated` и точный текст «без даты»
  - вызов `TaskService.list_undated(user)` — только `active`, `due_to IS NULL`, текущий пользователь
  - отправка через `TelegramSender.send_undated_list` или `send_empty_undated`
- `reminder/bot/handlers/callbacks.py`:
  - callback `delete:{task_id}` → `TaskService.delete_task()` → `TaskEvent(deleted)` + `send_deleted`
  - callback `assign_date:{task_id}` → подсказка «Ответь на это сообщение с датой» (карточка уже имеет `message_id` в `TaskEvent(undated_card_sent)`)
  - устаревшая кнопка → пересчитать актуальное состояние, не менять чужие/неактивные задачи

## TaskEvent

- При отправке каждой карточки → `TaskEvent(undated_card_sent, message_id=...)`
- При удалении → `TaskEvent(deleted)` (через `TaskService`, в одной транзакции со сменой статуса)

## Acceptance criteria

- `/undated` и «без даты» показывают одно и то же
- Показываются **все** активные задачи без даты (пагинация не реализуется — follow-up UC-F03)
- Каждая задача — отдельное сообщение с кнопками; после отправки — `TaskEvent(undated_card_sent, message_id)` (для TG-07)
- Удаление переводит задачу в `deleted` + создаёт `TaskEvent(deleted)`
- Чужие задачи недоступны
- Повторное нажатие старой кнопки не ломает состояние

## Ручная проверка разработчиком

- [ ] `/undated` при пустом списке — «Нет задач без даты»
- [ ] Создать 2–3 задачи без даты (voice/CORE-07) → `/undated` показывает все карточки с кнопками
- [ ] «Удалить» → задача `deleted` в БД, `TaskEvent(deleted)`, подтверждение в чате
- [ ] «Назначить дату» → подсказка «Ответь на это сообщение»; в БД есть `TaskEvent(undated_card_sent, message_id)`
- [ ] Повторное нажатие старой кнопки — не ломает состояние
