# TG-09: Тонкий voice handler

Тег: TG-09

Эпик: EPIC-03. Telegram-бот и пользовательские сценарии

Зависимости: TG-01, CORE-07

## Что реализуется

- `reminder/bot/handlers/voice.py` — handler на `voice` message
- Только wiring:
  1. `UserService.get_or_create_user(chat_id)`
  2. `TelegramSender.send_processing(chat_id)`
  3. `VoiceTaskCreationService.create_from_voice(user, voice)`
  4. `send_task_created` или `send_error` по `VoiceTaskResult`

## Acceptance criteria

- Handler не содержит бизнес-логики — только вызов CORE-07 + sender
- Успех и все ошибки отображаются пользователю понятным текстом (коды ошибок из TG-01)
- Подключён в dispatcher (TG-01)

## Ручная проверка разработчиком

- [ ] Отправить voice «напомни завтра в 15:00 позвонить врачу» → «Слушаю…» → подтверждение с задачей
- [ ] В БД: `Task(active)`, `TaskEvent(created)`, при наличии даты — `Reminder`
- [ ] Voice «без даты купить молоко» → задача без `Reminder`
- [ ] Слишком длинное voice → понятная ошибка, задача не создана
- [ ] `PARSER_BACKEND=mock` — работает без Yandex-ключей
