# TG-03: Реализовать `/start`

Тег: TG-03

Эпик: EPIC-03. Telegram-бот и пользовательские сценарии

Зависимости: TG-01, CORE-01

## Что реализуется

- `reminder/bot/handlers/start.py` — handler на команду `/start`
- Вызов `UserService.get_or_create_user(chat_id)`
- Ответ через `TelegramSender.send_welcome(chat_id)` (текст из TG-01)

## Acceptance criteria

- Первый `/start` создаёт `User` в PostgreSQL
- Повторный `/start` не создаёт дубль
- Пользователь получает приветственное сообщение (текст из TG-01)
- Handler не содержит логики кроме get_or_create + send_welcome
- `TaskEvent` не создаётся — действие над `User`, не над `Task`

## Ручная проверка разработчиком

- [ ] Первый `/start` — приветствие, в БД появился один `User`
- [ ] Повторный `/start` — то же приветствие, дубля `User` нет
- [ ] `chat_id` в БД совпадает с Telegram
