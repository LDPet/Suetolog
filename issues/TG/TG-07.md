# TG-07: Ввод даты через Reply + `TaskEvent` lookup

Тег: TG-07

Эпик: EPIC-03. Telegram-бот и пользовательские сценарии

Зависимости: TG-05, AI-04, CORE-03, DB-04

## Идея

Пользователь **отвечает (Reply)** на сообщение бота. Бот берёт `reply_to_message_id`, ищет `TaskEvent` с этим `message_id` и нужным `event_type` → получает `task_id` → переносит/назначает дату.

**Не нужно:** отдельная таблица `PendingDateInput`, поля `Task.list_message_id`, `Reminder.evening_question_message_id`.

## Зависимость от DB-04

В `TaskEvent` — поле `message_id` (nullable) + индекс. Типы событий для lookup:

| event_type | Кто создаёт | Когда |
| --- | --- | --- |
| `undated_card_sent` | TG-05 | Отправлена карточка задачи в `/undated` |
| `evening_question_sent` | BG-06 | Отправлен вечерний вопрос о переносе |
| `reminder_sent` | BG-01 | Уже есть — дополнить записью `message_id` |

## Lookup в handler

```python
event = TaskEventRepository.find_by_message_id(
    reply_to_message_id,
    event_types=['undated_card_sent', 'evening_question_sent'],
)
task_id = event.task_id
# → DateParserService.parse(text) → TaskService.assign_date() / reschedule()
```

## Два сценария

| Сценарий | Сообщение | TaskEvent |
| --- | --- | --- |
| `/undated` | Карточка задачи | `undated_card_sent` + `message_id` |
| Вечерний перенос | Вопрос «на когда перенести?» | `evening_question_sent` + `message_id` |

## Что реализуется

- `reminder/bot/handlers/date_reply.py` — handler на текст **с** `reply_to_message`
- `TaskEventRepository.find_by_message_id()` — поиск задачи по reply
- `TaskService.assign_date()` / `reschedule()` — создают `TaskEvent(date_set)` или `TaskEvent(rescheduled)` в той же транзакции
- Если текст без reply → подсказка, не парсим как дату

## TaskEvent

- Reply на `undated_card_sent` → после успеха → `date_set`
- Reply на `evening_question_sent` → после успеха → `rescheduled`

## Тексты

```
send_evening_question:
  «Задача «{title}» не выполнена.
   Ответь на это сообщение: на какую дату и время перенести?»

После callback assign_date:
  «Ответь на это сообщение с датой и временем.»

Без reply:
  «Чтобы назначить дату, ответь (Reply) на сообщение с задачей.»
```

## Acceptance criteria

- Reply на карточку `/undated` → `undated_card_sent` → дата назначается, `Reminder` создаётся
- Reply на вечерний вопрос → `evening_question_sent` → задача переносится, `TaskEvent(rescheduled)`
- Рестарт бота не ломает сценарий — `message_id` в PostgreSQL на `TaskEvent`
- Дата в прошлом отклоняется; текст без reply не обрабатывается как дата

## Ручная проверка разработчиком

- [ ] Reply на карточку `/undated` с «завтра в 15:00» → дата назначена, `Reminder` в БД, `TaskEvent(date_set)`
- [ ] Reply с датой в прошлом → отказ, задача не изменилась
- [ ] Текст без Reply в чат → подсказка, дата не назначается
- [ ] *(после BG-06)* Reply на вечерний вопрос → перенос, `TaskEvent(rescheduled)`
- [ ] Перезапустить `runbot` → reply всё ещё работает (lookup по `TaskEvent.message_id`)

## Поток (вечерний перенос)

```
BG-06: отправил вопрос msg_id=100
       → TaskEvent(task_id=42, event_type=evening_question_sent, message_id=100)

Юзер:  Reply «завтра в 15:00», reply_to_message_id=100

TG-07: TaskEvent WHERE message_id=100 → task_id=42 → reschedule
```

**Реакции ✅/❌** — по-прежнему через `Reminder.message_id` (TG-08), не через TaskEvent. Два разных механизма: реакция vs reply.
