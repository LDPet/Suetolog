# TG-07: Ввод даты через Reply + lookup по `TaskEvent`

Тег: TG-07

Эпик: EPIC-03. Telegram-бот и пользовательские сценарии

Зависимости: TG-05, TG-11, AI-04, CORE-03, DB-04

## Идея

Пользователь **отвечает (Reply)** на карточку задачи бота. Handler берёт `reply_to_message.message_id`, находит `TaskEvent` → `task_id` → парсит дату → `set_due_date` или `reschedule`.

**Не нужно:** `PendingDateInput`, `Task.list_message_id`, lookup через `Reminder.message_id`.

## Кто пишет `message_id` (TG-11)

| `event_type` | Когда |
| --- | --- |
| `undated_card_sent` | `/undated` |
| `reminder_sent` | due reminder (BG-01) |
| `digest_card_sent` | утренний дайджест (BG-05) |
| `evening_question_sent` | вечерний вопрос (BG-06) |

`TaskEvent.message_id` — **unique**. Одно сообщение Telegram → одна задача.

## Как по Reply получить task

```
1. message.reply_to_message — если None → подсказка, выход
2. event = TaskEventRepository.find_by_message_id(reply_to_message.message_id)
3. event.event_type ∈ CARD_EVENT_TYPES — иначе отказ
4. task_id = event.task_id
5. user = UserService.get_or_create_user(...)
6. parsed = DateParser.parse_date(message.text)
7. task.due_to is None → TaskService.set_due_date → TaskEvent(date_set)
   иначе              → TaskService.reschedule  → TaskEvent(rescheduled)
8. send_date_confirmed
```

```python
CARD_EVENT_TYPES = {
    TaskEvent.EventType.UNDATED_CARD_SENT,
    TaskEvent.EventType.REMINDER_SENT,
    TaskEvent.EventType.DIGEST_CARD_SENT,
    TaskEvent.EventType.EVENING_QUESTION_SENT,
}

event = TaskEventRepository.find_by_message_id(
    message.reply_to_message.message_id,
)
task_id = event.task_id
```

## Что реализуется

- `reminder/bot/handlers/date_reply.py` — текст **с** `reply_to_message`
- Фильтр: не команды, не voice, не callback
- Ошибки: `TaskNotFoundError`, `PermissionError`, `TaskStateError`, `TaskDateInPastError`, parser errors → текст пользователю
- Подключение в dispatcher (до generic echo handler)

## TaskEvent после успеха

| Было `due_to` | Метод | Новый event |
| --- | --- | --- |
| `NULL` | `set_due_date` | `date_set` |
| есть | `reschedule` | `rescheduled` |

## Тексты

```
Без reply:
  «Чтобы назначить или изменить дату, ответь (Reply) на сообщение с задачей.»

Неизвестная карточка:
  «Не нашёл задачу по этому сообщению.»

После успеха:
  send_date_confirmed (TG-01)
```

## Исключения

| Ситуация | Поведение |
| --- | --- |
| Текст без Reply | Подсказка, парсер не вызываем |
| `message_id` не в TaskEvent | Отказ |
| Задача done/deleted/чужая | Отказ через TaskService |
| Дата в прошлом | Отказ |
| Рестарт бота | Работает — lookup в PostgreSQL |

## Acceptance criteria

- Reply на undated/reminder/digest/evening карточку → дата назначена или перенесена
- Lookup **только** через `TaskEvent.find_by_message_id`
- Без Reply — подсказка, дата не меняется
- Дата в прошлом отклоняется

## Ручная проверка

- [ ] Reply на `/undated` карточку «завтра в 15:00» → `date_set`, Reminder при точном времени
- [ ] Reply на reminder «послезавтра в 18» → `rescheduled`
- [ ] Текст без Reply → подсказка
- [ ] Reply на `send_welcome` → отказ
- [ ] Перезапуск `runbot` → reply на старую карточку работает
- [ ] *(после BG-05/06)* Reply на digest/evening карточки
