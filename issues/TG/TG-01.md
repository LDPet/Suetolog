# TG-01: aiogram + `TelegramSender` + скачивание voice

Тег: TG-01

Эпик: EPIC-03. Telegram-бот и пользовательские сценарии

Зависимости: APP-01

Поглощает: TG-02, AI-01

## Перед началом разработки (блокер тикета)

- Создать **тестового бота** через [@BotFather](https://t.me/BotFather) в Telegram
- Получить `TELEGRAM_BOT_TOKEN`, прописать в `.env` / `config/settings.py` (не коммитить в git)
- Проверить, что бот отвечает на `/start` в личке (хотя бы echo или заглушка) — до полной реализации handlers

## Что реализуется

### Инфраструктура бота

- `python manage.py runbot` — запуск long polling
- `reminder/bot/dispatcher.py` — создание `Bot`, `Dispatcher`, подключение роутеров
- `reminder/bot/sender.py` — класс `TelegramSender`
- Чтение `TELEGRAM_BOT_TOKEN` из `config/settings.py`
- Интерфейс sender'а мокируется в тестах

### Скачивание файлов через Telegram API (бывший AI-01)

- `reminder/bot/telegram_files.py` — класс `TelegramFileDownloader` (или `download_voice(file_id) -> Path`)
- Скачивание OGG Opus по `file_id` через Telegram File API (`bot.get_file` → download)
- Проверка `VOICE_MAX_DURATION_SEC` и `VOICE_MAX_SIZE_BYTES` из settings **до** скачивания (по метаданным voice из update)
- Сохранение во временный файл; удаление в `finally` после обработки
- Без вызовов Yandex — только Telegram API; CORE-07 и AI-02 используют готовый temp-файл

> Скачивание voice — часть Telegram-слоя, не AI: это работа с Telegram File API, как и отправка сообщений через `TelegramSender`.

## Методы `TelegramSender`

| Метод | Назначение | Возвращает |
| --- | --- | --- |
| `send_text(chat_id, text)` | Произвольный текст | `message_id` |
| `send_welcome(chat_id)` | Приветствие `/start` | `message_id` |
| `send_processing(chat_id)` | Статус обработки voice | `message_id` |
| `send_task_created(chat_id, task)` | Подтверждение создания задачи | `message_id` |
| `send_error(chat_id, error_code)` | Ошибка пользователю | `message_id` |
| `send_undated_list(chat_id, tasks)` | Список задач без даты + inline-кнопки | `message_id` |
| `send_reminder(chat_id, task)` | Точечное напоминание | `message_id` |
| `send_evening_question(chat_id, task)` | Вечерний вопрос о переносе | `message_id` |
| `send_date_confirmed(chat_id, task)` | Подтверждение назначения даты | `message_id` |
| `send_deleted(chat_id, task_title)` | Подтверждение удаления | `message_id` |
| `send_empty_undated(chat_id)` | Нет задач без даты | `message_id` |

## Черновые тексты сообщений

```
send_welcome:
  «Привет! Я голосовой напоминальщик.
   Отправь голосовое сообщение с задачей — назови дело, дату и время.
   Если дата неизвестна — скажи «без даты».
   Команды: /undated — задачи без даты.»

send_processing:
  «Слушаю…»

send_task_created (с датой):
  «✅ Задача создана: {title}
   📅 {due_to в формате «15 июля, 15:00»}»

send_task_created (без даты):
  «✅ Задача создана: {title}
   Дата не указана. Назначить: /undated»

send_error (коды):
  voice_too_long    → «Голосовое слишком длинное. Попробуй короче.»
  voice_too_large   → «Файл слишком большой.»
  stt_empty         → «Не расслышал. Повтори голосом.»
  parser_failed     → «Не понял задачу. Попробуй сформулировать проще.»
  date_in_past      → «Дата уже прошла. Укажи будущую дату.»
  generic           → «Что-то пошло не так. Попробуй ещё раз.»

send_undated_list (на каждую задачу — отдельное сообщение):
  «📋 {title}
   [Назначить дату] [Удалить]»
  callback_data: assign_date:{task_id}, delete:{task_id}

send_empty_undated:
  «Нет задач без даты.»

send_reminder:
  «⏰ Напоминание: {title}»

send_evening_question:
  «Задача «{title}» не выполнена.
   Ответь на это сообщение: на какую дату и время перенести?»

send_date_confirmed:
  «📅 Дата назначена: {title} — {due_to}»

send_deleted:
  «🗑 Задача «{title}» удалена.»
```

## Acceptance criteria

- Тестовый бот создан в BotFather, токен в `.env.example` (placeholder) и локальном `.env`
- `python manage.py runbot` запускает бота локально без ошибок
- `TelegramSender` не содержит бизнес-логики — только форматирование и отправка
- Методы `send_reminder` и `send_evening_question` возвращают `message_id`
- `send_undated_list` отправляет inline-кнопки с `callback_data`, содержащим `task_id`
- Sender и `TelegramFileDownloader` подменяются моками в тестах
- Короткий voice скачивается по `file_id` → путь к temp-файлу
- Слишком длинный/большой voice отклоняется до скачивания и до вызова Yandex
- Temp-файл удаляется при успехе и ошибке

## Ручная проверка разработчиком

- [ ] `python manage.py runbot` — бот стартует без ошибок
- [ ] Бот виден в Telegram, отвечает на тестовое сообщение (echo/заглушка достаточно)
- [ ] Отправить короткое voice — в логах видно скачивание, temp-файл появился и удалился
- [ ] Отправить слишком длинное voice — отклонено до Yandex (если handler уже подключён) или видна проверка лимита в логе
