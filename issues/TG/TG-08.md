# TG-08: Реакции ✅ и ❌

Тег: TG-08

Эпик: EPIC-03. Telegram-бот и пользовательские сценарии

Зависимости: TG-01, CORE-03, CORE-05

## Правило: `TaskEvent`

| Действие | `event_type` | `message_id` |
| --- | --- | --- |
| Реакция ✅ | `completed` | null |
| Реакция ❌ | `cancelled` | null |

Событие пишет **сервисный слой** при смене статуса задачи.

## Что реализуется

- `reminder/bot/handlers/reactions.py` — handler на `message_reaction` update
- Поиск reminder: `ReminderService.find_by_message(chat_id, message_id)`
- ✅ → `TaskService.mark_done()` → `TaskEvent(completed)`
- ❌ → `TaskService.mark_cancelled()` → `TaskEvent(cancelled)`
- Повторные реакции игнорируются — повторный `TaskEvent` не создаётся

## TaskEvent

- Первая ✅ → `completed` (в транзакции с `Task.status = done`)
- Первая ❌ → `cancelled` (в транзакции с `Task.status = cancelled`)

## Acceptance criteria

- Реакция находится по `chat_id + message_id` сохранённому при отправке reminder
- Первая ✅ → `Task.status = done` + `TaskEvent(completed)`
- Первая ❌ → `Task.status = cancelled` + `TaskEvent(cancelled)`
- Повторная или изменённая реакция не меняет финальный статус
- После `done`/`cancelled` будущие reminders по задаче не отправляются

## Ручная проверка разработчиком

- [ ] *(нужен reminder — BG-01 или тестовый в БД)* Поставить ✅ → `Task.status=done`, `TaskEvent(completed)`
- [ ] Поставить ❌ на другом reminder → `cancelled`, `TaskEvent(cancelled)`
- [ ] Повторная реакция — статус не меняется
- [ ] Сверить `chat_id + message_id` reminder с реакцией в БД
