Зависимости: AI-01..AI-06

## Описание

Нужно реализовать часть pipeline `voice/text -> parser -> TaskCreationService`. Используй `tz/VOICE_PIPELINES.md`, `tz/ARCHITECTURE.md`, `tz/use_cases.md` и настройки из `config/settings.py`; внешние Yandex-зависимости должны быть изолированы так, чтобы их можно было мокировать в тестах.

В рамках тикета нужно: проверены лимиты voice, успешный STT, пустой transcript, timeout/5xx, 401/403, невалидный JSON, удаление временного файла. Это должно быть реализовано в границах MVP из `decomp.md` и не должно тянуть в этот тикет соседние задачи, если они явно вынесены в зависимости или следующие эпики.

Особое внимание удели ошибкам внешних сервисов, таймаутам, лимитам voice-сообщений и тому, чтобы секреты не попадали в логи. Для локальной разработки должен оставаться путь с моками без реальных Yandex credentials.

## Критерии приемки

- Проверены лимиты voice, успешный STT, пустой transcript, timeout/5xx, 401/403, невалидный JSON, удаление временного файла.
- Реализация согласована с `README.md`, `tz/ARCHITECTURE.md` и профильными ТЗ-файлами, указанными в описании проекта.
- В коде, тестах, логах и документации нет реальных секретов, Telegram/Yandex токенов или приватных credentials.

## Что проверить вручную/автоматически

- Покрыть unit-тестами успешный путь и ошибки внешнего API или parser-валидации.
- Проверить, что реальные ключи и токены не попадают в логи, README и fixtures.
- Проверить работу с моками без доступа к Yandex API.
- Для voice-сценариев проверить очистку временных файлов после успеха и ошибки.
