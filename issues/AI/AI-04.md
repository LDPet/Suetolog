# AI-04: Parser даты для назначения/переноса

Тег: AI-04

Эпик: EPIC-04. AI и голосовой пайплайн

Результат: Свободный русский ввод → datetime в `DEFAULT_TIMEZONE`; дата в прошлом отклоняется

Зависимости: AI-03

## Технологии и контекст

- **Отличие от AI-03:** здесь пользователь пишет **только дату/время** (Reply-текст), не целую задачу. Отдельный prompt / метод `parse_date()`, не `parse_task()`.
- **YandexGPT** — тот же клиент, что AI-03; меняется system prompt и формат ответа (одно datetime, не полная задача).
- **Reply в Telegram** — пользователь отвечает на **конкретное сообщение** бота; TG-07 находит задачу по `TaskEvent.message_id`. Parser **не** делает lookup — только «завтра в 15:00» → datetime.
- **Timezone** — все даты в `Europe/Moscow` (`DEFAULT_TIMEZONE`), timezone-aware datetime.
- Consumer: **TG-07**, use cases [UC-06](tz/use_cases.md), [UC-12](tz/use_cases.md).

## Границы тикета

**Входит:** `parse_date(text) → datetime`.

**Не входит:** parser задачи (AI-03), Reply handler (TG-07), STT.

## Что сделать

1. Метод разбора только даты/времени — отдельный prompt.
2. Переиспользовать YandexGPT-клиент из AI-03.
3. Timezone-aware datetime MSK.
4. Past date → ошибка (коды как AI-03/06).
5. Unit-тесты с mock API.

## Кривые кейсы — учесть

| Ситуация | Ожидание |
| --- | --- |
| Дата в прошлом («вчера», «15 мая» если уже прошло) | `date_in_past` |
| Только время «в 15:00» без дня | Интерпретация через `now` — зафиксируй в prompt/тестах |
| «Через 2 часа» | Future datetime от `now` |
| Reply на чужую/устаревшую карточку | TG-07 отклоняет — parser может не вызываться |
| Тот же prompt, что для полной задачи | **Не использовать** — путает title и date |
| `PARSER_BACKEND=mock` | Простой mock для demo — на твоё усмотрение |
