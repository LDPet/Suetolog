# AI-03: YandexGPT parser задачи

Тег: AI-03

Эпик: EPIC-04. AI и голосовой пайплайн

Результат: `parse_task()` → `ParsedTaskInput`; JSON Schema; дата в прошлом отклоняется

Зависимости: APP-01, CORE-02, AI-06

## Технологии и контекст

- **YandexGPT / Foundation Models** — managed LLM в Yandex Cloud. Ты шлёшь system + user prompt, получаешь текст; в MVP нужен **structured output** — JSON по схеме, не свободный текст.
- **JSON Schema в prompt** — описываешь модели формат ответа (`title`, `due_to`, …). После ответа **валидируешь** JSON сам (не полагайся только на «обещание» модели).
- **`now` + timezone** — в prompt передаёшь текущие дату/время MSK, чтобы «завтра», «в понедельник» разрешались однозначно.
- **Temperature ~0.1** — низкая, чтобы ответы были стабильнее. Настройки — `config/settings.py`.
- **Интерфейс AI-06** — `TaskParser.parse_task(text) -> ParsedTaskInput`; подключается через `get_parser()` при `PARSER_BACKEND=yandex`.
- Документация: `tz/VOICE_PIPELINES.md` § YandexGPT parser, § JSON Schema, § Правила маппинга.

## Границы тикета

**Входит:** разбор **полной задачи** из русского текста; интеграция в `get_parser()`.

**Не входит:** STT (AI-02), parser даты для Reply (AI-04), Task (CORE-03), Telegram.

## Что сделать

1. HTTP-клиент Foundation Models — endpoint в VOICE_PIPELINES.
2. Parser implements `TaskParser` из AI-06.
3. System prompt — **составь сам** (требования в VOICE_PIPELINES).
4. Валидация по JSON Schema + примеры из VOICE_PIPELINES.
5. Ошибки: пустой title, past date, invalid JSON → коды как в AI-06.
6. `raw_text` = входной transcript.

## Примеры входного текста

| Текст / transcript | Ожидание |
| --- | --- |
| `Купить корм для кота` | title, `due_to=null`, без выдуманного description |
| `Напомни завтра в 15:00 позвонить врачу` | title + future `due_to` |
| `Отправить отчёт сегодня до 18, добавить цифры продаж` | title, description, `due_to` |
| `Каждый понедельник в 9 проверить финансы` | `repeat_type=weekly` (due_to может быть null) |
| `Напомни в пятницу` | Дата без времени — см. маппинг VOICE_PIPELINES (Reminder?) |
| `Сделать то же что вчера` / явная прошлая дата | `date_in_past` |
| `Привет как дела` | `parser_failed` — не задача |

**В чате *(voice, prod path)*:** те же фразы голосом → после STT parser должен дать тот же результат.

## Кривые кейсы — учесть

| Ситуация | Ожидание |
| --- | --- |
| GPT вернул JSON в markdown-блоке ` ``` ` | Вырезать/отклонить — только чистый JSON |
| Две задачи в одной фразе | Одна задача (главная) или `parser_failed` — зафиксируй в prompt |
| Модель «додумала» дату, которой не было | Prompt: null для отсутствующих полей |
| `due_to` только дата, без времени | Task создаётся; точечный Reminder — по правилам CORE-03 |
| `due_to` в прошлом (в т.ч. «сегодня 8:00» в 18:00) | `date_in_past` |
| Timeout / 5xx YandexGPT | 1 retry, затем ошибка пользователю |
| Невалидный JSON / пустой title | `parser_failed` |
| Секреты или полный текст в логах | **Запрещено** |
