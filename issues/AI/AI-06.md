# AI-06: mock-parser (волна 0, до AI-03)

Тег: AI-06

Эпик: EPIC-04. AI и голосовой пайплайн

Результат: `PARSER_BACKEND=mock|yandex`; `get_parser()`; default mock в `.env.example`

Зависимости: APP-01, CORE-02

## Зачем этот тикет

**Волна 0** — до AI-03. Нужен переключаемый parser, чтобы CORE-07 и TG-09 могли прогнать голосовой сценарий **без Yandex-ключей**.

## Технологии и контекст

- **Parser** — компонент, который из русского текста (transcript после STT) делает структуру `ParsedTaskInput` (title, due_to, …). В prod это YandexGPT (AI-03); в dev — mock.
- **Фабрика `get_parser()`** — переключатель по env (`PARSER_BACKEND`). Паттерн «strategy»: один интерфейс, разные реализации. CORE-07 не знает, mock или Yandex — только вызывает `get_parser().parse_task(...)`.
- **Mock-parser** — не ML. Простые правила (подстроки, regex), чтобы тесты и demo были **детерминированными**. Настоящий NLP — в AI-03.
- Документация: `tz/VOICE_PIPELINES.md` (контракт полей), `tz/ARCHITECTURE.md` (слой AI отдельно от CORE и TG).

## Границы тикета

**Входит:** общий интерфейс parser → `ParsedTaskInput`, mock-реализация, фабрика `get_parser()` по `PARSER_BACKEND`.

**Не входит:** HTTP к Yandex (AI-03), STT (AI-02), создание Task (CORE-03), Telegram (TG-09).

## Что сделать

1. Ввести интерфейс parser (один метод: текст → `ParsedTaskInput`). Контракт — CORE-02.
2. Реализовать **mock-parser**: предсказуемый, достаточный для unit-тестов и demo. Не NLP — простые правила, которые **ты** выберешь. Главное — стабильность тестов.
3. Фабрика `get_parser()`: `mock` → mock; `yandex` → реальный parser (AI-03, когда готов).
4. Settings + `.env.example`: default `PARSER_BACKEND=mock`.
5. Общие коды ошибок parser (`parser_failed`, `date_in_past`) — чтобы CORE-07 обрабатывал их так же, как ошибки Yandex-parser.

| Голосовое | Что должно произойти |

## Кривые кейсы — учесть

| Ситуация | Ожидание |
| --- | --- |
| `PARSER_BACKEND=yandex`, AI-03 ещё не готов | Понятная ошибка конфигурации, не silent fail |
| Пустой/пробельный transcript попал в parser | `parser_failed`, Task не создаётся |
| В тексте несколько задач подряд | Mock может взять первую — в prod это решает prompt AI-03 |
| «Завтра» без времени | `due_to` с датой или null — согласуй с CORE-03 (Reminder только при точном времени) |
| Переключили env без перезапуска worker | Достаточно перезапуска `runbot` / django process |

## Acceptance criteria

- `PARSER_BACKEND=mock` — pipeline без Yandex credentials
- `get_parser()` переключает backend
- Mock: сценарии **с датой** и **без даты**
- Unit-тесты mock + фабрики
