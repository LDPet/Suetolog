# DB-01: Окружение PostgreSQL и модель User

Тег: DB-01

Эпик: EPIC-01. Модель данных и PostgreSQL

Результат: PostgreSQL подключён; `make migrate`; заготовки `repositories/` и `tests/`; модель User с `chat_id`, `telegram_user_id`; UserRepository; фикстура `user`; тесты

Зависимости: APP-01, APP-04

## Описание

Первый тикет EPIC-01: **инфраструктура БД + таблица пользователей**. Блокирует DB-02..DB-04.

### Часть 1 — инфраструктура

1. **Settings** — PostgreSQL из `.env` / `docker-compose.yml` (APP-04): `DATABASES`, `INSTALLED_APPS` + `reminder`, `TIME_ZONE` (`Europe/Moscow`).

2. **Makefile:**
   ```makefile
   migrate:
   	python manage.py migrate
   ```
   README: `docker compose up -d` → `make migrate` → `make test`.

3. **Каталоги:** `reminder/repositories/`, `reminder/tests/conftest.py` с `@pytest.mark.django_db`.

4. **Транзакционный контракт** (README или `repositories/README.md`):
   - ORM только в `repositories/`, не в handlers и CORE;
   - связанные записи — через `transaction.atomic()` в сервисах; validation — в CORE.

### Часть 2 — модель User

| Поле | Telegram | Назначение |
|---|---|---|
| `chat_id` | `message.chat.id` | Lookup бота; unique |
| `telegram_user_id` | `message.from_user.id` | ID пользователя; unique |
| `created_at` | — | auto |

Отдельная модель (не `AbstractUser`) — без login/password (`tz/ARCHITECTURE.md`).

### Repository

`reminder/repositories/users.py` — методы студент выводит из `tz/use_cases.md` (UC-01), `issues/CORE/CORE-01.md`. Mapping → use cases в PR. `get_or_create_user` — в UserService (CORE-01).

### Фикстура и тесты

- фикстура `user` в conftest;
- model: create; duplicate `chat_id` → `IntegrityError`;
- transaction: rollback в `atomic()`;
- ≥1 тест на каждый repository-метод.

### Разблокирует

**CORE-01**

## Критерии приемки

- `make migrate` на чистой БД; `manage.py check` OK.
- Infra: `repositories/`, conftest, транзакционный контракт в README.
- User: оба ID unique; миграция; UserRepository; фикстура; тесты проходят.
- Секретов нет.

## Что проверить

- `make migrate` дважды подряд.
- Shell: User create, повторный `chat_id` падает.
