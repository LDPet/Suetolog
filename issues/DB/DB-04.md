# DB-04: Модель TaskEvent и atomic bundle

Тег: DB-04

Эпик: EPIC-01. Модель данных и PostgreSQL

Результат: Audit-события; TaskEventRepository; atomic Task + Reminder + TaskEvent; фикстура; transaction-тесты

Зависимости: DB-03

## Описание

История действий по задаче (`tz/ARCHITECTURE.md`). Зависит от DB-03, чтобы все три связанные модели были готовы для atomic bundle.

### Поля

FK → Task; `event_type` (enum); `message_id` (nullable, Telegram message_id для Reply-lookup); `created_at`.

### Типы событий

`created`, `reminder_sent`, `undated_card_sent`, `evening_question_sent`, `completed`, `cancelled`, `rescheduled`, `date_set`, `deleted`, `digest_sent` (опционально).

### Lookup

`TaskEventRepository.find_by_message_id(message_id)` — для TG-07 (Reply на карточку undated или вечерний вопрос).

### Индексы

`(task, created_at)`.

### Repository

`reminder/repositories/task_events.py` — методы из use cases, CORE-03/04.

### Atomic bundle

Helper в `repositories/` (например `bundles.py`):

- `transaction.atomic()`: Task + опциональный Reminder + TaskEvent;
- вызов из TaskCreationService (CORE-03), не из handlers;
- validation даты — в сервисе до вызова.

Инвариант: при ошибке БД partial writes нет (`tz/use_cases.md`).

### Фикстура и тесты

- `task_event`;
- create + `event_type`;
- atomic commit и rollback;
- ≥1 тест на repo-метод.

### Разблокирует

**CORE-03**, **DOC-02**.
