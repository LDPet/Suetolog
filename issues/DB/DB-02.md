# DB-02: Модель Task

Тег: DB-02

Эпик: EPIC-01. Модель данных и PostgreSQL

Результат: Модель задачи по ТЗ; FK на User; статусы; индексы; TaskRepository; фикстура `task`; тесты

Зависимости: DB-01

## Описание

Согласовать с `tz/tables.md`, `tz/ARCHITECTURE.md`.

### Поля

`title`, `description`, `due_to` (nullable), `repeat_type`, `repeat_interval`, `status`, `created_at`, FK → User (`CASCADE`).

### Статусы

`active`, `done`, `cancelled`, `deleted` — логическое удаление. Default: `active`. Нужен `__str__`.

### Индексы

`(user, status)`, `(user, due_to)`.

### Repository

`reminder/repositories/tasks.py` — методы из use cases, `CORE-03`. Mapping в PR.

### Фикстура

`task` — активная задача; использует `user` из DB-01.

### Тесты

create + FK; invalid `status`; ≥1 тест на repo-метод.

## Критерии приемки

- Поля и статусы по ТЗ; индексы в миграции.
- TaskRepository; фикстура; тесты; mapping в PR.

## Что проверить

- Shell: Task для User.
- `makemigrations --check`.
