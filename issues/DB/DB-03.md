# DB-03: Модель Reminder

Тег: DB-03

Эпик: EPIC-01. Модель данных и PostgreSQL

Результат: Модель напоминания; FK на Task; индексы; ReminderRepository; фикстура `reminder`; тесты

Зависимости: DB-02

## Описание

Единица отправки в Telegram и привязки реакции ✅/❌ (`tz/MAILING_PIPELINES.md`).

### Поля

`reminder_time`, `sent_time` (nullable), `reaction` (nullable), `message_id`, FK → Task (`CASCADE`).

Опционально для BG-F01 (post-MVP): `status`, `locked_at`, `attempts`.

### Индексы

`reminder_time`, `sent_time`, `message_id`, `task`.

### Repository

`reminder/repositories/reminders.py` — из use cases UC-09..11, `CORE-05`. Mapping в PR.

### Фикстура

`reminder` — будущий `reminder_time`, привязан к `task`.

### Тесты

create + FK; ≥1 тест на repo-метод.

### Разблокирует

**CORE-05** (с DB-02..DB-04 для полного домена).

## Критерии приемки

- Поля и индексы; ReminderRepository; фикстура; тесты; mapping в PR.

## Что проверить

- Due reminders query через repository (`reminder_time`, `sent_time IS NULL`).
