# APP-04: Поднять `docker-compose.yml` для PostgreSQL и Redis

Тег: APP-04

Эпик: APP

Результат: локальное окружение поднимает PostgreSQL и Redis одной командой. Параметры подключения задокументированы для последующего добавления в `config/settings.py`.

Зависимости: APP-01

## Описание

Нужно добавить `docker-compose.yml` для инфраструктуры MVP: PostgreSQL как основное хранилище бизнес-данных и Redis как broker для Celery. Это должно соответствовать архитектуре из `tz/ARCHITECTURE.md`, требованиям к хранению из `tz/tables.md`, пользовательским сценариям из `tz/use_cases.md` и ограничениям из `decomp.md`, где Redis не должен быть единственным местом хранения бизнес-состояния.

Compose должен поднимать два сервиса: PostgreSQL и Redis. Для PostgreSQL нужно настроить database/user/password через переменные окружения или безопасные local defaults, открыть порт для локальной разработки и добавить volume, чтобы данные не терялись при рестарте контейнеров. Redis должен быть доступен приложению и будущим Celery worker/beat.

На этом этапе `config/settings.py` может быть ещё пустым — достаточно зафиксировать параметры подключения в `docker-compose.yml`, `.env.example` и README, чтобы их позже перенесли в Django settings в профильных тикетах.

Не нужно в этом тикете реализовывать модели, миграции или Celery tasks. Важно создать устойчивый инфраструктурный baseline, от которого смогут зависеть DB-тикеты, Celery-тикеты и локальная разработка Telegram/AI flow.

## Критерии приемки

- В репозитории есть `docker-compose.yml`.
- Команда `docker compose up` поднимает PostgreSQL и Redis.
- Параметры подключения (host, port, database, user, password, Redis URL) задокументированы в `.env.example` и/или README.
- Для PostgreSQL настроен volume, и данные не теряются при обычном рестарте контейнера.
- Redis используется как инфраструктурный сервис для broker/cache-сценариев, а не как единственное хранилище бизнес-состояния.
- README или локальная документация содержит команду запуска compose-окружения.

## Что проверить вручную/автоматически

- Запустить `docker compose up` и убедиться, что оба сервиса становятся healthy или успешно принимают подключения.
- Перезапустить контейнер PostgreSQL и убедиться, что volume не пересоздаётся случайно.
- Сравнить параметры подключения в compose и `.env.example`.
